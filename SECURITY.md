# Security And Sensitive-Data Reporting

Please report security, privacy, source-safety, or culturally sensitive display issues through the repository issue tracker, or by contacting the maintainer if public disclosure would expose sensitive material.

Report especially:

- committed secrets, tokens, credentials, or private service identifiers;
- local paths, private notes, unreleased review files, or internal data accidentally exposed in public files;
- records that appear to include restricted, suppressed, non-public, or culturally sensitive material in a public export;
- misleading framing that presents public records as verified supernatural events;
- crawler, robots, or source-use concerns for public materials.

This project is a public-text research archive. Public discoverability is not treated as unrestricted permission, and Indigenous-related records require special care around terminology, publicness, cultural sensitivity, source voice, and display mode.

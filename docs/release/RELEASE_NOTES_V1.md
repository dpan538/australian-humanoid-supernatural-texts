# Release Notes V1

Australian Public Text Archive of Supernatural Humanoid Narratives and Encounters is a typed, provenance-aware public research archive for tracing how humanoid or humanoid-adjacent supernatural figures appear in Australian public texts.

The project documents public source records. It does not verify supernatural claims.

## Current Corpus

Current generated release baseline:

- Public records: 3809.
- Mapped public records: 1206.
- Date span: 1825-2026.
- Source organisations with public records: 33.
- Registered source organisations audited: 42.

## Source Model

The archive separates primary or strong public sources, secondary/context sources, discovery-only sources, and excluded/restricted materials. Public metadata and public pages are treated as evidence of a public record or lead, not permission to reproduce restricted cultural material.

## Map Caveat

Each public map flag is a verified display location for one public record. It is not proof of an event, a habitat, a population distribution, or cultural authority. Broad-region, state-only, culturally sensitive, and unclear records may remain unmapped.

## Ethics Caveat

Indigenous-related and culturally sensitive records require caution around publicness, terminology, source voice, sensitivity, and display mode. Public availability does not override cultural restrictions or community control.

## Known Limitations

- The corpus is not complete.
- It is not peer reviewed.
- It is not a representative distribution of real-world events.
- Source concentration remains high for Internet Archive and Australian Yowie Research.
- Some metadata and Wikipedia-derived records remain review flags rather than claims of strong source authority.
- The frontend is desktop-first and small-screen usable, not a dedicated mobile product.

## Next Work

- Continue source diversification.
- Reduce discovery-only and metadata-only ambiguity.
- Improve mobile-specific interaction design.
- Add clearer public review queues for source concentration and sensitivity.
- Continue validating map eligibility after every data export.

# Data Health V2

V2 assesses archive quality, not whether a supernatural claim is true.

Allowed analysis statuses:

- `analysis_ready`
- `display_ready_reviewed`
- `display_ready_unreviewed`
- `review_required`
- `lead_only`
- `excluded`

Automated scripts must never assign `analysis_ready`. That status requires explicit human review.

Shared health dimensions include source traceability, stable identifier, publicness, rights/access status, classification confidence, source mediation, transcription quality, human review status, ethics review status, and duplicate/variant resolution.

Type-specific checks vary. Encounter accounts require event specificity, event date quality, event location quality, morphology evidence, witness/reporting context, and source proximity. Traditional or spirit-person narratives require public attribution, narrator/collector/author context, public community/language context where appropriate, source authority, sensitivity/publicness review, and reuse status.
